# Regras de evidência

Todo achado de agente deve ser **verificável**. Vale para revisões, análises e
relatórios de qualquer skill ou subagente deste template.

- Citar `arquivo:linha` (clicável).
- Mostrar o trecho relevante ou o comando que comprova (ex.: saída de teste/linter).
- Distinguir **fato** (observado) de **hipótese** (a confirmar) — nomeie qual é qual.
- Sinalizar explicitamente lacunas de validação: "não consegui verificar X porque Y".
- **Sem evidência verificável → rebaixar a severidade ou marcar como "a investigar".**
  Nunca reportar como CRÍTICO/ALTO algo que não foi comprovado.
