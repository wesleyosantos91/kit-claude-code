# /adr

Delegue ao subagente `tech-writer` a criação de um ADR para a decisão descrita
abaixo, usando o template e a numeração sequencial definidos no próprio agente
(`agents/tech-writer.md`).

Antes de escrever, o tech-writer deve ler o contexto real do repositório
(código, ADRs existentes em `docs/architecture/adr/`) — nunca inventar
alternativas que não foram consideradas.
